# みおり

> **宙ぶらりんを、宙ぶらりんのまま大切にする場所。**

問いの変形を、解決せずに記録する場所。

普通のメモアプリは「答え」を残す。
このアプリは「揺れている最中」と「変形の軌跡」を残す。

形にしないまま置いておく場所。

公開URL：[https://oga-lab.github.io/miori/](https://oga-lab.github.io/miori/)

---

## 4つの場所

| 場所 | 役割 |
|---|---|
| **みお** | 日々の問いや揺れの記入場所 |
| **みおあと** | 流れたみおの一覧 |
| **おり** | 問いが5つセットで層に分かれる場所 |
| **みおつくし** | みおの温度・矛盾などが、ランダムで浮かび、立ち上がる場所 |

水の上・水の中・水の底・水のしるべ。

---

## 時代の中での位置づけ

AI時代が加速する中で、「すぐに答えを得る」「効率を上げる」が優先される世界が来る。
その世界では、人間の曖昧さを保持し、自分で問いと向き合う時間が、少しずつ死んでいく。

みおりは、AIと深く付き合いながらも、
人間がゆっくり自分を見つめ、社会や倫理、哲学の中で問いを生み、
それをすぐに解決せず、人生の中でいろんなものに触れたり、体感しながら、
ゆっくり答えを見つけていく——そのための手がかりアプリ。

答えをくれるAIに対する、答えをくれない場所。
効率に対する、非効率の聖域。

## 技術構成

- フレームワーク不使用（生のHTML/CSS/JS）
- IndexedDBラッパーに `idb` ライブラリをCDNから利用
- フォント：明朝体（OS標準）
- カラー：和紙の白 + 墨
- LLM（生成AI）未使用

### ローカルで動かす

Service Worker は `file://` では動かないので、簡易サーバーで起動する。

```bash
# Python 3
python3 -m http.server 8000

# Node.js
npx serve
```

ブラウザで `http://localhost:8000` を開く。

### GitHub Pagesで公開

1. GitHubに新しいリポジトリを作る（例: `miori`）
2. このフォルダの中身を全部pushする
   ```bash
   git init
   git add .
   git commit -m "初期コミット"
   git branch -M main
   git remote add origin https://github.com/[ユーザー名]/miori.git
   git push -u origin main
   ```
3. GitHub上で **Settings → Pages**
4. **Source: Deploy from a branch** → **main / (root)** を選択 → Save
5. 数分後、`https://[ユーザー名].github.io/miori/` で公開される

### スマホのホーム画面に追加

公開URLをスマホで開く：

**iPhone (Safari)**
1. URL を Safari で開く
2. 共有ボタン → **ホーム画面に追加**

**Android (Chrome)**
1. URL を Chrome で開く
2. メニュー → **ホーム画面に追加** または通知の「インストール」

ホーム画面のアイコンから開くと、ブラウザのバーが消えてアプリのように動作する。

### データの保存場所

すべてのデータはブラウザの **IndexedDB** に保存される。
端末の中だけにあり、外部サーバーには一切送信されない。

ブラウザのデータを消去するとデータも消えるので注意。
